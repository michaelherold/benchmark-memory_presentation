One of the causes

def concat_javascript_sources(buf, source)
  if buf.bytesize > 0
    buf << ";" unless string_end_with_semicolon?(buf)
    buf << "\n" unless buf.end_with?("\n")
  end
  buf << source
end

def string_end_with_semicolon?(str)
  i = str.size - 1
  while i >= 0
    c = str[i]
    i -= 1
    if c == "\n" || c == " " || c == "\t"
      next
    elsif c != ";"
      return false
    else
      return true
    end
  end
  true
end

How can we speed this up?

SEMICOLON_REGEX = /(;\s*\z|\A\s*\z)/.freeze

def string_end_with_semicolon?(str)
  return false if str.size.zero?
  
  str === SEMICOLON_REGEX
end

Hypothesis: Reverse regex

SEMICOLON_REGEX = /(\A\s*;|\A\s*\z)/.freeze

def string_end_with_semicolon?(str)
  return false if str.size.zero?
  
  str.reverse === SEMICOLON_REGEX
end

benchmark-memory

Measuring the memory performance of your code

(in a benchmarking style)

Profiling: memory_profiler

  • Total memory allocated and retained
  • Number of strings allocated and retained
  • Number of objects allocated and retained
  • MRI 2.1+ only

Model: benchmark-ips

Benchmark.ips do |x|
  x.report("test 1") { "str" + "append" }
  x.report("test 2") { "str" << "append" }
  
  x.compare!
end
